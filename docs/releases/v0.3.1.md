# vibe-council v0.3.1 — dogfood hardening for the decision-memory / context loop

The **hardening** release. v0.3.0 shipped the local-first decision-memory / context-pack loop;
v0.3.1 **uses that loop on real work** and fixes the rough edges that surfaced — **no new command
surface**. Everything stays **deterministic and local-first**: no MCP, no rolling summaries, no
token-aware tokenizer, no vector retrieval, no LLM eval, no hosted/sync. Based on and crediting
[`karpathy/llm-council`](https://github.com/karpathy/llm-council).

The full changelog is in
[`CHANGELOG.md`](https://github.com/EfeAydinalp/vibe-council/blob/master/CHANGELOG.md).

> **No commercial-clearance claim.** License/provenance remains an unresolved **"Question 0"**
> commercial gate (see `docs/plans/license-and-provenance-resolution.md`); no `LICENSE` is added.

## Upgrade / check

```sh
git pull
vibe --version        # vibe-council 0.3.1
vibe lint --redaction # scan public docs for leaks (no tokens spent)
vibe doctor           # check your provider setup (no tokens spent)
```

No new runtime dependencies. The v0.3 commands call **no model / API / network**.

## Highlights

- **`decisions promote` won't promote a scaffold** — an all-`TODO` draft is refused; core sections
  (Decision, Rationale, and Consequences/Next actions) must carry meaningful content.
- **Curated filenames** — promoted records follow the `YYYY-MM-DD-slug.md` convention (no more
  `DEC-….md`).
- **Better `--from-run` extraction** — maps a review's verdict, rationale, alternatives, risks/
  consequences, and next actions into the matching draft sections (deterministic; unmatched sections
  keep `TODO`s). Long slugs are capped/sanitized.
- **`context check` is 21/21 on the real repo** — context packs now carry an explicit human-review
  promotion boundary, and the default char budget was bumped (12000 → 14000) so the pack keeps its
  core signals.
- **CLI UX polish** — clearer `lint --redaction` pass/fail verdict, a `context build` budget hint,
  a `context check` advisory-miss count, and clearer `decisions` help around local drafts / human
  review / no auto-stage.
- **Public-safe dogfood notes** — `docs/dogfood/v0.3.1-notes.md` records the findings that drove
  these fixes.

## The v0.3 loop (unchanged shape)

```text
vibe decisions new --from-run <local-review.md>
# edit / review / redact the draft
vibe decisions promote <draft.md>        # writes a curated YYYY-MM-DD-slug.md
vibe decisions lint
vibe context build
vibe context check                       # 21/21 on a curated repo
vibe context export claude-code
```

Supporting: `vibe lint --redaction` (scan public docs) and `vibe operator status` (local status).

## Bug fixes

- `decisions promote` no longer accepts placeholder-only core drafts.
- `decisions promote` writes `YYYY-MM-DD-slug.md` instead of `DEC-….md`.
- `decisions new --from-run` maps obvious review sections into the draft.
- Long extracted slugs are capped/sanitized.
- Context packs include the human-review promotion boundary; `context check` reaches 21/21.
- Context-pack default char budget bumped 12000 → 14000 so core signals survive trimming.

## Safety boundaries

- **Raw `.council/` stays local/gitignored** — reviews, drafts, context packs, Claude Code exports,
  and operator status are never committed.
- **Generated packs/exports are local by default**; the builder/export **refuse to write under
  `docs/`** unless `--allow-docs`, and **block on critical redaction findings**.
- **`docs/decisions/*.md` remains the curated source of truth**; promotion is **human-reviewed** and
  never auto-commits.
- **`CLAUDE.md` is never modified** by the export.
- **License/provenance remains "Question 0"** — no commercial offering, no `LICENSE`, no clearance
  claim.

## Known limitations

- Context budget is still **character-based** (not a real tokenizer) — bumped to 14000 for the MVP;
  a token-aware budget remains future work. The real-repo `context check` guard will flag if the
  curated set outgrows the budget again.
- The context-quality harness is **deterministic, not an LLM eval**.
- Extraction is heuristic and **path-only** (`--from-run <path>`).
- MCP, rolling summaries, vector/hybrid retrieval, operator event log, dashboards, mobile, and
  hosted/sync are **not** included.

## Next roadmap

Prerequisite-gated, after v0.3.1: a standalone **v0.4** design — likely **read-only MCP export** of
the pack and **Claude Code workflow** improvements — opened only after this release publishes. Then
**rolling summaries** and a **token-aware budget**; **vector/hybrid retrieval** only if plain
retrieval proves insufficient; and — only after the **license/provenance** gate clears and demand is
validated — a separate **hosted/team/sync** layer. See
[`docs/plans/track-based-roadmap.md`](../plans/track-based-roadmap.md).

It's an early `0.x` release — expect breaking changes between minor versions. The council's output is
**advice to filter, not authority** — you own the decision.
