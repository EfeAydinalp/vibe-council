# vibe-council v0.3.0 — local-first decision memory + curated project context

The **decision-memory / context** release. On top of v0.2's multi-provider core, vibe-council now has
an end-to-end, **deterministic, local-first** loop for capturing decisions and turning them into
agent-ready project context — with a redaction guard across the workflow. No LLM summarization, no
vector retrieval, no MCP, no hosted/sync. Based on and crediting
[`karpathy/llm-council`](https://github.com/karpathy/llm-council).

The full changelog is in
[`CHANGELOG.md`](https://github.com/EfeAydinalp/vibe-council/blob/v0.3.0/CHANGELOG.md).

> **No commercial-clearance claim.** License/provenance remains an unresolved **"Question 0"**
> commercial gate (see `docs/plans/license-and-provenance-resolution.md`); no `LICENSE` is added.

## Upgrade / check

```sh
git pull
vibe --version        # vibe-council 0.3.0
vibe lint --redaction # scan public docs for leaks (no tokens spent)
vibe doctor           # check your provider setup (no tokens spent)
```

No new runtime dependencies. The v0.3 commands call **no model / API / network**.

## Highlights

- **Redaction guard** — `vibe lint --redaction`: a stdlib-only scanner for public docs (API keys,
  private keys, secret assignments, per-user local paths, dated `.council/` artifact paths,
  `.obsidian/` workspace); `--strict`, masked output, JSON via `decisions lint`/`context` flows.
- **Curated decision CLI** over `docs/decisions/*.md`: `vibe decisions list` (`--tag`/`--status`),
  `show` (path-traversal guarded), `new` (template), `lint` (frontmatter, stable headings, duplicate
  ids, broken links, redaction).
- **Draft extraction** — `vibe decisions new --from-run <review.md>`: deterministic extraction (no
  LLM) of a **local** draft into gitignored `.council/decisions/drafts/`.
- **Safe promotion** — `vibe decisions promote <draft.md>`: validates + writes a curated record into
  `docs/decisions/` (`--dry-run`/`--force`); never auto-commits; never reads raw run logs.
- **Context pack** — `vibe context build`: deterministic pack from curated decisions + `STATUS.md`
  with a character budget; `vibe context check`: a deterministic quality harness (not an LLM eval);
  `vibe context export claude-code`: wrap the pack as a Claude Code-friendly local file.
- **Operator status** — `vibe operator status` / `set` / `clear`: a tiny local status surface
  (`.council/operator/status.json`).
- **License/provenance "Question 0"** — an engineer's checklist + decision recording the commercial
  gate.

## The v0.3 loop

```text
vibe decisions new --from-run <local-review.md>
# edit / review / redact the draft
vibe decisions promote <draft.md>
vibe decisions lint
vibe context build
vibe context check
vibe context export claude-code
```

## Safety boundaries

- **Raw `.council/` stays local/gitignored** — reviews, decisions, drafts, context packs, Claude Code
  exports, and operator status are never committed.
- **Generated context packs/exports are local by default**; the builder/export **refuse to write
  under `docs/`** unless `--allow-docs`, and **block on critical redaction findings**.
- **`docs/decisions/*.md` remains the curated source of truth**; promotion is **human-reviewed**.
- **`CLAUDE.md` is never modified** by the export.

## Known limitations

- Context budget is **character-based** (not a real tokenizer) for the MVP.
- The context-quality harness is **deterministic, not an LLM eval**.
- Extraction is heuristic and **path-only** (`--from-run` id lookup is later).
- **License/provenance is unresolved** — no commercial offering, no `LICENSE`, no clearance claim.
- MCP, hosted/sync, dashboards, mobile, and custom remote transport are **not** included.

## Verification checklist

```sh
python -m unittest discover -s tests -t .   # full suite
vibe lint --redaction                       # 0 critical
vibe decisions lint                         # curated records pass
vibe context build && vibe context check    # pack builds + passes (100%)

```

## Next roadmap

Prerequisite-gated, after v0.3.0: **MCP read-only export** of the pack; **rolling summaries** and a
**token-aware budget**; **vector/hybrid retrieval** only if plain retrieval proves insufficient;
**skill/council packs**; and — only after the **license/provenance** gate clears and demand is
validated — a separate **hosted/team/sync** layer. See
[`docs/plans/track-based-roadmap.md`](https://github.com/EfeAydinalp/vibe-council/blob/v0.3.0/docs/plans/track-based-roadmap.md).

It's an early `0.x` release — expect breaking changes between minor versions. The council's output is
**advice to filter, not authority** — you own the decision.
