# Prepare v0.3.1 (dogfood hardening for the decision-memory / context loop)

## Context

v0.3.0 shipped the local-first decision-memory / context loop. The v0.3.1 cycle then dogfooded
that loop ([dogfood notes](../dogfood/v0.3.1-notes.md)) and landed three small, PR-sized hardening
fixes: decision-CLI rough edges, the `context check` human-review signal, and a CLI UX pass. This
record captures the decision to cut **v0.3.1** as a hardening release — docs/metadata only, no new
command surface.

## Decision

Publish **v0.3.1** as the *dogfood hardening* release:

- **Bump the canonical version to `0.3.1`** in `backend/__init__.py` and `pyproject.toml`;
  `vibe --version` reports `vibe-council 0.3.1`. (`uv.lock` is a generated lockfile left as-is, as
  it was for v0.3.0; regenerating it is out of scope and risks dependency changes.)
- **Release prep is docs/metadata only:** CHANGELOG `[0.3.1]` section (Added / Fixed / Changed /
  Safety / Deferred), `docs/releases/v0.3.1.md`, README release-status + the v0.3 loop, STATUS, and
  the agent brief.
- **No git tag, no GitHub Release in this PR** — the maintainer cuts `v0.3.1` after the release PR
  merges and it is explicitly requested.
- The untracked planning doc `docs/plans/v0.3.1-hardening-and-dogfood.md` is **left local/untracked**
  (its root-cause hypothesis was superseded by the dogfood finding; the committed dogfood notes +
  release notes + decision records are the accurate record).

## Rationale

- v0.3.1 is a hardening milestone; everything stays deterministic and local-first, with no new
  surface to document or stabilize.
- Keeping prep to docs/metadata avoids behavior risk right before a tag.
- Tag/release as a deliberate, separate human step preserves the no-surprise-publish posture.

## Alternatives considered

- **Fold the hardening into a delayed v0.4** — rejected; the fixes are complete and shippable now,
  and v0.4 (MCP / Claude Code workflow) deserves its own design after v0.3.1 publishes.
- **Tag inside this PR** — rejected; tagging/release is a separate, explicitly-requested step.
- **Commit the v0.3.1 hardening plan doc** — rejected; it carries a superseded hypothesis and would
  confuse readers.
- **Bump `uv.lock`** — rejected; it is generated and was already left at its prior value for v0.3.0.

## Consequences

- The repo reports `0.3.1` and ships v0.3.1 notes; the loop is hardened and `context check` is 21/21.
- MCP, rolling summaries, token-aware budget, vector retrieval, LLM eval, operator event log,
  dashboard/mobile, and hosted/sync remain deferred.
- License/provenance remains the commercial "Question 0".

## Next actions

- Merge this release-prep PR, then (when requested) tag `v0.3.1` and publish the GitHub Release.
- After release: open a standalone v0.4 design (likely read-only MCP / Claude Code workflow).
