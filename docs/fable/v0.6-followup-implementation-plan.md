## 1. Current status

  validation. No side effects; strict unknown-key rejection; server-minted fields reject on sight.

  <file | ->` — validated proposals become Task + pending ApprovalRequest + pending Action; payload

  allowlist (still the fixed verification/status set), dependencies (stdlib-only), version (`0.5.2`).

## 4. Budget policy (binding)

- **Fable = technical lead / architect only.** Fable reviews plans, resolves architecture conflicts,
  and designs new major phases. It does **not** implement routine PRs.
- **Opus/Sonnet implement normal PRs** (everything in §3, and v0.6.1+ once scoped).
- **Do not re-engage Fable** until: a new major phase needs design (v0.6.1+ scoping, v0.7, v0.8), a
  major architecture review is explicitly requested, or a serious blocker/security conflict appears.
- **No broad Fable repo scans for routine PRs.** Each PR reads only the files its prompt lists.
- Council review budget: cheap for routine/docs, balanced for security-relevant diffs, full only for
  major security/architecture questions (none expected in §3).

## 6. Stop conditions (halt and ask a human; do not push through)

- Any **security-invariant conflict** ([03-security-invariants.md](03-security-invariants.md)).
- A **broad unexpected diff** (change sprawls beyond the PR's listed files/scope).
- **Executor/trust/panel changes outside the stated scope** of the current PR.
- **Dependency churn** (any new runtime dep; any `uv.lock` change outside an intentional release-prep
  self-version sync).
- **Command allowlist change** of any kind.
- **Version/tag/release changes** not explicitly instructed.
- Any **private/runtime artifact touched** (`.council/`, payloads, `.env`, private plans, secrets).
