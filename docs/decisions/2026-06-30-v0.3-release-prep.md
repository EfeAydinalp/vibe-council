# Prepare v0.3.0 (local-first decision memory + curated project context)

## Context

The v0.3 decision-memory / context-pack work is implemented end-to-end: redaction guard, curated
decision CLI, draft extraction, safe promotion, context build/check/export, operator status, and the
license/provenance checklist. This record captures the decision to cut **v0.3.0** as the next
milestone.

## Decision

Publish **v0.3.0** as the *local-first decision memory + curated project context* release:

- **Bump the canonical version to `0.3.0`** in `backend/__init__.py` and `pyproject.toml`; `vibe
  --version` reports `vibe-council 0.3.0`.
- **Release prep is docs/metadata only:** CHANGELOG `[0.3.0]` section (Added / Changed / Safety /
  Deferred), `docs/releases/v0.3.0.md`, README release-status + the v0.3 loop, STATUS, and agent brief.
- **No git tag and no GitHub Release in this PR** — those are cut by a maintainer after the release PR
  merges and is explicitly requested (per `docs/release-checklist.md`).
- **No commercial-clearance claim;** license/provenance remains "Question 0", and no `LICENSE` is added.

## Rationale

- The decision-memory / context loop is complete and dogfoodable; a tagged milestone gives a clean
  reference point before the next (MCP / summaries / budget) work.
- Keeping tag/release out of the PR matches the project's release process and avoids premature tags.

## Alternatives considered

- **Tag in this PR** — rejected; tagging/release is a separate, explicit maintainer step.
- **Bundle MCP / rolling summaries into v0.3.0** — rejected; those are deferred, prerequisite-gated.
- **Claim commercial readiness** — rejected; license/provenance is unresolved.

## Consequences

- v0.3.0 is the decision-memory / context milestone; everything stays deterministic and local-first.
- Generated `.council/` artifacts remain gitignored; `docs/decisions/*.md` stays source-of-truth.
- MCP, rolling summaries, token-aware budget, vector retrieval, packs, hosted/sync, and dashboards/
  mobile remain **deferred**.

## Next actions

- Merge this release-prep PR, then (when requested) tag `v0.3.0` and publish the GitHub Release.
- After release: consider MCP read-only export or a token-aware budget / rolling summaries.
