# v0.3.1 CLI UX consistency pass

## Context

The v0.3.1 dogfood ([dogfood notes](../dogfood/v0.3.1-notes.md)) flagged small CLI rough edges
that are not bugs but hurt clarity: `vibe lint --redaction` printed counts without a clear
pass/fail verdict; `vibe context build` trimmed to budget with a terse warning; `vibe context
check` did not summarize advisory misses; and `vibe decisions` help under-stated the
promotion/human-review boundary. None change behavior — they are output/help polish.

## Decision

A small, conservative output/help pass (in `backend/cli.py`, stdlib-only, no behavior change):

- **Redaction lint verdict.** One clear line:
  `redaction lint passed: 0 critical, N warning(s) (<scope>)` or
  `redaction lint FAILED: X critical, Y warning(s) (<scope>)`. Warning details and the
  advisory-vs-`--strict` note are unchanged; secrets stay masked.
- **Context build budget note.** When budget trimming happens, print the current char budget
  and suggest `--max-chars` (no tokenizer, no default change beyond PR #50's bump).
- **Context check.** Append an advisory-miss count to the score line; per-check listing and
  the JSON report are unchanged.
- **Decisions help.** Document the boundary: `new --from-run` writes only a LOCAL gitignored
  draft; `promote` requires meaningful reviewed content and never auto-stages/commits.
- **Docs.** README now reflects post-#49/#50 behavior (promoted `YYYY-MM-DD-slug.md`
  filenames, broadened `--from-run` mapping, placeholder-rejecting `promote`).

## Rationale

- A single verdict line makes the redaction guard's result obvious at a glance while keeping
  the detailed findings.
- The budget note turns an opaque trim into an actionable hint.
- Help text is where the human-review boundary is most useful to a first-time user.
- Output/help-only changes are low-risk and keep behavior and feature scope stable.

## Alternatives considered

- **Leave output as-is** — rejected; the verdict and budget messages were genuinely unclear in
  dogfood.
- **Redesign the `context check` per-check listing** — rejected as over-scope; a one-line
  advisory-miss summary is enough.
- **Make `lint --redaction` scan untracked files too** — deferred; tracked-only is intentional,
  now clarified in output scope rather than changed.

## Consequences

- Clearer CLI output/help; no behavior, dependency, or provider change; JSON output stable.
- New CLI-output tests (`tests/test_cli_ux.py`).
