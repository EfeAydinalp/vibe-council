# Accept the council correction: v0.5 is the AI Council Workbench MVP

## Context

A roadmap-correction council pass (`vibe review` + `vibe full`, `balanced`) reviewed everything built
since v0.2.0 and found: **strong infrastructure, no user-visible product loop.** The raw input,
outputs, and local synthesis stayed under gitignored `.council/` (source material only). This record
accepts the correction and sets the v0.5 direction; the full plan is in
[v0.5 Workbench MVP](../plans/v0.5-workbench-mvp.md).

## Decision

- **Accept the council correction.** The v0.2–v0.4 work (provider abstraction, decision memory,
  context packs, redaction, Claude Code export, operator status, read-only MCP) is **foundation, not
  waste** — leverage it, stop expanding it.
- **v0.5 becomes the Workbench MVP, not more MCP/context infrastructure.** It builds one minimal
  **vertical slice** — task → visible stages → **audited approval** → safe execution → logged — for
  a single workflow: **"safe repo changes with approval."**
- **First implementation PR is the runtime data models + a `.council/runtime/` JSON store**
  (stdlib-only, gitignored) — `Task` / `Stage` / `ApprovalRequest` / `ApprovalDecision` / `Action` /
  `AuditResult`. Everything else (orchestrator, panel, auditor) flows from that.
- **The Approval Auditor ships with the first approval UI but is advisory only.** **Deterministic
  guards are the real security boundary** — path allow/deny, command allowlist/templates,
  secret-file patterns, change-size limits, and cloud-egress consent (redaction before any provider
  call). The AI adds a risk label + a short human-readable approval rewrite, and may flag/escalate,
  but never gates.
- **Single-model + audited approval is the MVP;** wiring the existing multi-model council as a
  discussion stage is **optional/last**.
- **Mobile, voice, and personalization are deferred** (v0.6+). Hosted/team/sync and remote approval
  transport stay post-1.0 behind a security design.
- **Existing v0.2–v0.4 work becomes the foundation:** MCP = read-only knowledge source;
  decisions/context = memory; operator status = the panel's minimal status surface; `.council/` =
  the local runtime/generated workspace (curated `docs/decisions/` remains the committed
  source-of-truth).

## Roadmap / messaging correction

- **Near-term product name: "AI Council Workbench."**
- **"local-first AI project OS" is long-term / internal only** — an architectural north star, **not**
  near-term external messaging until post-v1.0.
- Corrected sequence: **v0.5** Workbench MVP (vertical slice) → **v0.6** mobile (LAN web) → **v0.7**
  voice → **v0.8** personalization → **v0.9** polish/security → **v1.0** sponsor/demo-ready.

## Rationale

- The biggest risk is momentum/product drift, not technology; a tangible, interactive loop proves
  value and reuses the infrastructure already built.
- A vertical slice validates the core interaction (see → decide → act) in one release instead of
  five to six.
- Anchoring security in deterministic guards (not an LLM) keeps the approval boundary trustworthy;
  the auditor improves UX without becoming a false security control.

## Alternatives considered

- **Keep building horizontal infra (more MCP/context/docs)** — rejected; it deepens the "engine with
  no steering wheel" problem.
- **Panel-only v0.5, approvals in v0.6** — rejected; a progress bar you can't act on has little
  value; the loop must ship together.
- **Treat the Approval Auditor as the security boundary** — rejected; LLMs are stochastic and
  prompt-manipulable; deterministic guards are the boundary.
- **Include multi-model council discussion in the v0.5 slice** — deferred; single-model + audited
  approval is the true MVP; wiring the council is optional/last and must not block the slice.
- **External "AI project OS" positioning now** — rejected as premature.

## Consequences

- v0.5 shifts from infrastructure to a user-visible Workbench loop; the next PR is the runtime data
  models + store, not another MCP/context surface.
- Mobile/voice/personalization/hosted and new MCP/CLI surfaces are frozen until the panel exists.
- License/provenance remains the commercial "Question 0".

## Next actions

- Next implementation PR: **runtime data models + `.council/runtime/` JSON store** (stdlib-only,
  gitignored, no server).
- Then: orchestrator/state machine → deterministic trust boundary → hybrid Approval Auditor → local
  panel with integrated approval loop.
