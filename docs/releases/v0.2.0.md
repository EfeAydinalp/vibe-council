# vibe-council v0.2.0 — multi-provider OpenRouter + local Ollama

The **multi-provider** release. vibe-council is still a local-first AI "council" CLI, but
it no longer hard-wires a single provider: a provider abstraction now sits under the
council, OpenRouter remains the default, and you can run entirely **locally with Ollama**.
A new `vibe doctor` checks your setup without spending tokens. Based on and crediting
[`karpathy/llm-council`](https://github.com/karpathy/llm-council).

The full changelog is in
[`CHANGELOG.md`](https://github.com/EfeAydinalp/vibe-council/blob/v0.2.0/CHANGELOG.md).

## Highlights

- **Provider abstraction** — a minimal `Provider` seam (`ChatRequest` / `ChatResult`) with
  an OpenRouter adapter. OpenRouter behavior is unchanged.
- **Provider selection** — pick a provider with `VIBE_PROVIDER` (default `openrouter`);
  unsupported values fail clearly before any model call.
- **Local Ollama provider** — `VIBE_PROVIDER=ollama` talks to a local Ollama server
  (`/api/chat`, non-streaming). No API key; **loopback-only** `OLLAMA_HOST` (SSRF-safe);
  never fabricates a dollar cost.
- **`VIBE_OLLAMA_MODEL`** — map the preset's OpenRouter-style model IDs to a local model
  you've pulled, without redesigning presets.
- **`vibe doctor`** — provider diagnostics (key/host validation, OpenRouter model-list and
  Ollama `/api/tags` reachability, local model list + override availability). Runs **no
  inference**; supports `--offline`.
- **Provider-aware cost messaging** — `--usage` / `--max-cost` name the active provider and
  are honest when a provider reports no cost.
- **Fixed:** `full` mode no longer crashes when a ranking model returns empty/`None`.
- **Removed:** the unused upstream web UI subsystem and FastAPI/Uvicorn dependencies.

## Upgrade / check

```sh
git pull
vibe --version        # vibe-council 0.2.0
vibe doctor           # check your provider setup (no tokens spent)
vibe doctor --offline # config checks only, no network
```

## OpenRouter (default)

```sh
# .env: OPENROUTER_API_KEY=sk-or-v1-<your key>
vibe review --preset balanced --file plan.md --yes --usage
```

## Ollama (local)

```sh
# Run a local Ollama server (https://ollama.com) and pull a model first.
export VIBE_PROVIDER=ollama
export VIBE_OLLAMA_MODEL=llama3.1     # a model you've `ollama pull`ed
vibe doctor                           # verifies host + server + local models
vibe review --preset cheap --file plan.md --yes
```

## Known limitations

- **Ollama users should set `VIBE_OLLAMA_MODEL`** — presets still carry OpenRouter-style
  model IDs; provider-specific preset config is future work.
- **Local Ollama reports no billing cost**, so `--max-cost` cannot be enforced for Ollama
  runs (cost is never fabricated).
- **MCP, personas/advisors, app/TUI, and community** features are still future work.
- **License/provenance cleanup is ongoing** — parts of the backend still descend from the
  unlicensed upstream; no `LICENSE` is added yet.

It's an early `0.x` release — expect breaking changes between minor versions. The
council's output is **advice to filter, not authority** — you own the decision.
