# Prepare v0.4.0 (read-only MCP / Claude Code workflow)

## Context

The v0.4 read-only MCP track is implemented and dogfooded: the contract (`vibe mcp contract`), the
dependency-free read layer (`vibe mcp inspect`), in-memory context pack + health reads, and a stdlib
JSON-RPC **stdio transport** (`vibe mcp serve --stdio`), plus context-pack critical-section
stabilization and setup/dogfood docs. This record captures the decision to cut **v0.4.0** as the
read-only MCP release.

## Decision

Publish **v0.4.0** as the *read-only MCP / Claude Code workflow* release:

- **Bump the canonical version to `0.4.0`** in `backend/__init__.py` and `pyproject.toml`;
  `vibe --version` reports `vibe-council 0.4.0`. (`uv.lock` is a generated lockfile left as-is, as
  for v0.3.x; no dependency change.)
- **Release prep is docs/metadata only:** CHANGELOG `[0.4.0]` section (Added / Changed / Safety /
  Verification / Deferred), `docs/releases/v0.4.0.md` (tag-pinned absolute links), README
  release-status + MCP commands, STATUS, and the agent brief.
- **MCP transport is stdlib JSON-RPC over stdio — no `mcp` SDK dependency.** The exposed surface is
  limited to **status, curated decisions, the context pack, and context health**; context reads are
  **in-memory / no-write**.
- **No git tag, no GitHub Release in this PR** — the maintainer cuts `v0.4.0` after the release PR
  merges and it is explicitly requested.

## Rationale

- v0.4.0 is the read-only MCP milestone; everything stays deterministic and local-first, with no new
  authority and no runtime dependency.
- Keeping prep to docs/metadata avoids behavior risk right before a tag.
- Tag/release as a deliberate, separate human step preserves the no-surprise-publish posture.

## Alternatives considered

- **Add the `mcp` SDK for "official" transport before releasing** — rejected; the stdlib transport
  works and stays dependency-light; an optional extra remains a later option.
- **Ship write-capable MCP / standalone resources in v0.4.0** — rejected; deferred by scope.
- **Tag inside this PR** — rejected; tagging/release is a separate, explicitly-requested step.
- **Bump `uv.lock`** — rejected; generated and unchanged (no dependency change).

## Consequences

- The repo reports `0.4.0` and ships v0.4.0 notes; the read-only MCP workflow is documented and
  dogfooded.
- Write-capable MCP, remote approval, hosted/sync, broader protocol/client integration, standalone
  resources, token-aware budget, vector retrieval, LLM eval, and rolling summaries remain deferred.
- License/provenance remains the commercial "Question 0".

## Next actions

- Merge this release-prep PR, then (when requested) tag `v0.4.0` and publish the GitHub Release.
- After release: verify exact Claude Code config; consider optional standalone resources or an
  optional `mcp` SDK extra only if a real need appears.
