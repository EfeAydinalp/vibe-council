# Publish v0.2.0 (multi-provider milestone)

## Context

After the provider-abstraction arc (OpenRouter seam, local Ollama, `vibe doctor`,
provider-aware cost) and release-prep work, vibe-council was ready for its first
multi-provider release.

## Decision

Publish **v0.2.0** as the multi-provider milestone, gated on:

- **118 tests passing** (stdlib `unittest`, CI on Ubuntu/macOS/Windows).
- **`v0.2.0` tag verified**; `vibe --version` reports `vibe-council 0.2.0`.
- Release notes documenting highlights **and known limitations**.

## Rationale

- The provider work was complete and stable, with no OpenRouter behavior regression.
- A tagged milestone gives a clean reference point for downstream planning (decision
  memory, commercial review) without those efforts destabilizing the core.

## Alternatives considered

- **Bundle more features into v0.2.0** (decision-memory tooling, MCP) — rejected; ship the
  provider milestone clean and keep new work downstream.
- **Delay the release for licensing resolution** — rejected; licensing is a separate,
  longer track and should not block a stable feature milestone.

## Consequences

- **v0.2.0 is the multi-provider milestone**; the default OpenRouter path is unchanged.
- **Known limitations carried forward:** Ollama users must set `VIBE_OLLAMA_MODEL`; local
  Ollama reports no billing cost (so `--max-cost` cannot be enforced for it); MCP, personas,
  app/TUI, and community features are still future work; license/provenance cleanup is ongoing.
- **Next work should not destabilize the core** — keep the council review UX stable.

## Next actions

- Treat decision memory and the commercial review as **downstream** of core stability.
- Default real runs to the `balanced` preset; avoid `premium`/`full` unless asked.
